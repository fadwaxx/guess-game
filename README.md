# 🎯 خمّن العنصر — لعبة تخمين أونلاين لشخصين (المرحلة 1: النسخة الأساسية)

هذه هي **المرحلة الأولى فقط** من المشروع، كما طُلب: إنشاء الغرفة، دخول لاعبين، توزيع
العناصر السرية، مزامنة الدور والمؤقت، والتخمين. **البطاقات المميزة، خيار عشوائية الفئة
كل جولة، المؤثرات الصوتية/البصرية، وحفظ البيانات في قاعدة بيانات دائمة ستُضاف في
المراحل التالية** بعد التأكد من أن هذا الأساس يعمل بشكل صحيح.

## لماذا هذه التقنيات؟

- **React + TypeScript + Tailwind + Vite**: تطوير سريع وواجهة متجاوبة بأقل قدر من الإعداد.
- **Node.js + Express + Socket.IO**: Socket.IO هو الخيار الأنسب للمزامنة اللحظية (الأدوار،
  المؤقت، التخمينات) بين لاعبين، وأبسط من إعداد WebRTC مباشر.
- **تخزين في الذاكرة (Phase 1) بدل قاعدة بيانات فورًا**: لأن حالة الغرفة مؤقتة أساسًا
  (مباراة تستمر دقائق)، فالتخزين في الذاكرة كافٍ الآن ويسهّل الاختبار السريع. في مرحلة
  لاحقة يمكن إضافة Supabase أو MongoDB لحفظ الإحصائيات التاريخية إن أردت، دون تغيير
  منطق اللعبة نفسه لأن `roomManager.ts` معزول تمامًا عن بقية الكود.

### مبدأ التصميم المهم: لا شيء مرتبط بـ"الحيوانات" في الكود

كل الوظائف (توزيع العناصر، التخمين، حفظ النتائج، اختيار الصور) تعمل فقط عبر
`categoryId` و`itemId` من `server/src/data/categories.ts`. فئة الحيوانات هي مجرد
بيانات مسجَّلة في `ALL_CATEGORIES`. لإضافة فئة جديدة مستقبلًا (بلدان، طعام، مشاهير...):

1. أنشئ ملفًا جديدًا في `server/src/data/` (مثل `countries.ts`) يصدّر مصفوفة عناصر
   بنفس شكل `GameItem` الموجود في `types.ts`.
2. أضفه إلى `ALL_CATEGORIES` في `categories.ts` بسطرين فقط.
3. لا حاجة لتعديل `gameLogic.ts` أو `socketHandlers.ts` أو أي مكوّن في الواجهة إطلاقًا.

النصوص في الواجهة تستخدم كلمة **"عنصر"** بشكل عام (وليس "حيوان")، إلا عند عرض بيانات
فئة الحيوانات نفسها حيث يظهر اسم العنصر الفعلي (مثل "أسد") كما هو منطقي.

## متطلبات التشغيل

- [Node.js](https://nodejs.org) الإصدار 18 أو أحدث (يتضمن npm).

## خطوات التشغيل على Windows

### 1) تثبيت Node.js
حمّل النسخة LTS من nodejs.org وثبّتها بالإعدادات الافتراضية، ثم تأكد بفتح
Command Prompt (cmd) وكتابة:
```
node -v
npm -v
```

### 2) فك ضغط المشروع وفتحه
فك ضغط المجلد `guess-game` في أي مكان، مثلاً `C:\guess-game`، ثم افتح Command Prompt
وانتقل إليه:
```
cd C:\guess-game
```

### 3) تشغيل الخادم (Server)
في نافذة Command Prompt أولى:
```
cd server
copy .env.example .env
npm install
npm run dev
```
سترى رسالة: `✅ الخادم يعمل على المنفذ 4000`. اترك هذه النافذة مفتوحة.

### 4) تشغيل الواجهة (Client)
افتح نافذة Command Prompt **ثانية** منفصلة:
```
cd C:\guess-game\client
copy .env.example .env
npm install
npm run dev
```
سيظهر رابط مثل `http://localhost:5173` — افتحه في المتصفح.

### 5) تجربة اللعبة بلاعبين
- افتح الرابط في نافذة متصفح، اكتب اسمك واضغط "إنشاء غرفة جديدة".
- انسخ رمز الغرفة (أو الرابط) وافتحه في نافذة متصفح أخرى (أو جهاز آخر على نفس الشبكة
  باستخدام عنوان IP للجهاز بدل localhost)، واكتب اسمًا آخر وانضم بالرمز.
- اضغط الطرفان زر "أنا جاهز" لتبدأ الجولة تلقائيًا.

## النشر (اختياري لاحقًا)

- **الخادم**: يمكن نشره مجانًا على [Render](https://render.com) (Web Service من نوع Node).
- **الواجهة**: يمكن نشرها مجانًا على [Vercel](https://vercel.com) أو [Netlify](https://netlify.com).
- بعد النشر، غيّر `VITE_SERVER_URL` في `.env` الخاص بالواجهة إلى رابط الخادم المنشور،
  و`CLIENT_ORIGIN` في `.env` الخاص بالخادم إلى رابط الواجهة المنشورة.

## ما تم اختباره منطقيًا في هذه المرحلة

- ✅ إنشاء غرفة والحصول على رمز قصير (5 أحرف/أرقام غير ملتبسة).
- ✅ انضمام لاعب ثانٍ عبر الرمز أو الرابط، ومنع دخول لاعب ثالث.
- ✅ عدم بدء اللعبة إلا بعد ضغط كلا اللاعبين "جاهز".
- ✅ توزيع عنصر عشوائي مختلف لكل لاعب من الفئة المختارة (حاليًا: الحيوانات فقط، والبنية
  جاهزة لأي فئة).
- ✅ كل لاعب يرى عنصر منافسه فقط، والعنصر الخاص به سري حتى في بيانات الشبكة (لا يُرسل
  إلا بعد `revealed = true`).
- ✅ مؤقت دور خادم-محوري (Server-authoritative): إذا انتهى الوقت، ينتقل الدور تلقائيًا
  حتى لو أُغلق المتصفح.
- ✅ مطابقة تخمين تتجاهل التشكيل، "أل" التعريف، اختلاف الهمزات، وتسمح بخطأ إملائي بسيط.
- ✅ مهلة 5 ثوانٍ بين كل تخمين وآخر لمنع التخمين العشوائي المتكرر.
- ✅ أفضل من 1/3/5 جولات، مع بدء اللاعب الآخر أول كل جولة تحقيقًا للعدالة.
- ✅ عند انقطاع لاعب: يُعلَم الطرف الآخر، وتُحذف الغرفة تلقائيًا بعد 5 دقائق دون عودته.

## محدودية معروفة في هذه المرحلة (ستُعالَج لاحقًا)

- إذا **حدّث اللاعب نفسه الصفحة (Refresh)**، سيحصل على `socket.id` جديد ولن يُتعرَّف عليه
  كنفس اللاعب تلقائيًا بعد. سنضيف "معرّف لاعب دائم" (playerId) في المرحلة القادمة لدعم
  العودة الفعلية بعد تحديث الصفحة أو انقطاع الإنترنت.
- لا توجد بعد بطاقات مميزة (الزرقاء/الخضراء)، ولا خيار "عشوائية الفئة كل جولة"، ولا
  مؤثرات صوتية، ولا حفظ دائم في قاعدة بيانات. هذه في المرحلة التالية.

## الخطوة التالية

بعد أن تجرّب هذه النسخة وتتأكد من عملها بشكل صحيح بين جهازين مختلفين، أخبرني لأبدأ
بإضافة: البطاقتين المميزتين (التبديل + الأسئلة الثلاثة)، دعم بقية الفئات (البلدان،
الطعام، الجمادات...)، المؤثرات الصوتية والبصرية، وتحسين التعامل مع الانقطاع/التحديث.

## تحديث المقدمة والخلفية
- تمت إضافة فيديو افتتاحي في `client/public/media/game-intro.mp4`.
- تمت إضافة الخلفية الثابتة في `client/public/media/game-background.png`.
- يظهر الفيديو مرة واحدة في كل جلسة متصفح باستخدام `sessionStorage`.
- عند انتهاء الفيديو يختفي تدريجيًا وتبقى الصورة نفسها خلفية للموقع.
- لإعادة تجربة المقدمة أثناء التطوير، افتح أدوات المطور ثم نفّذ:
  `sessionStorage.removeItem('guess-game-intro-seen')`
  وبعدها حدّث الصفحة.
